# Content Pillars for upsk.to Promotion

## Pillar 1: "The AI Instructor" (Product Demo)
Show what the AI teaching experience actually looks like.

**Formats:** Remotion videos, LinkedIn text posts, carousels
**Hooks:**
- "I pasted one line and my AI became my instructor"
- "Watch an AI teach system design to a beginner"
- "This is what adaptive teaching looks like"
- "My AI instructor remembered me from yesterday"

**Content ideas:**
- Screen recording of actual teaching session (anonymized)
- Before/after: same module, different students, different teaching approaches
- "Session 1 vs Session 5" showing how AI adapts
- Teaching transcript snippets that show the bouncer/passport analogies

## Pillar 2: "Evidence Over Grades" (Philosophy)
The scoring system that shows HOW you think.

**Formats:** Text posts, Remotion videos, carousels
**Hooks:**
- "Your GPA says nothing about how you think under pressure"
- "We don't give grades. We record evidence."
- "5 dimensions. Every score backed by what you actually did."
- "Employers want proof of thinking. Not certificates."

**Content ideas:**
- Show a real scoring output (anonymized): dimension scores + evidence quotes
- Compare: resume with GPA vs upsk profile with evidence
- "What a hiring manager sees" — show the employer view
- Post about why traditional testing is broken

## Pillar 3: "One Prompt" (Simplicity / Technical)
The absurd simplicity of the onboarding.

**Formats:** Remotion videos (terminal boot sequence), text posts
**Hooks:**
- "One prompt. That's the entire onboarding."
- "Paste this in Codex. You now have a bootcamp."
- "Zero signup forms. Zero credit cards. One prompt."
- "Your AI editor is now your instructor"

**Content ideas:**
- 15-second Remotion video: terminal typing the prompt, boot sequence, teaching starts
- "I replaced a 12-week bootcamp with one line of text"
- Technical breakdown of how skill.md works (for developer audience)
- Compare setup time: bootcamp signup (25 min) vs upsk (15 seconds)

## Pillar 4: "Built Different" (Architecture / Behind the Scenes)
How we built this — agentic, spec-driven, no BS.

**Formats:** Text posts, threads
**Hooks:**
- "I built an entire EdTech platform with zero lines of code written by me"
- "The teaching agent that teaches itself to teach better"
- "How we shipped 47 modules with Codex and Claude Code"
- "Our CI pipeline has more opinions than our developers"

**Content ideas:**
- Architecture deep-dive: agent does thinking, API does plumbing
- The spec-driven development story (ties to Ravi's SDD content)
- How the adaptive memory works across sessions
- Open source the scoring rubric

## Pillar 5: "For Colleges" (B2B)
Targeting placement officers and department heads.

**Formats:** LinkedIn text posts, direct outreach
**Hooks:**
- "Your placement team is running blind"
- "Evidence of thinking > placement test scores"
- "One day to set up. Students start with one prompt."
- "See how 200 students think, not just what they scored"

**Content ideas:**
- Dashboard mockup showing college analytics
- ROI calculation: cost of upsk vs traditional training programs
- "What a placement officer actually needs" — pain point post
- Case study format (even pre-launch: "here's what the data would show")

## Pillar 6: "Student Stories" (Social Proof)
Real teaching moments, real scores, real growth.

**Formats:** Remotion videos, carousels, text posts
**Hooks:**
- "This student went from 'I don't know' to explaining stateless auth in 3 minutes"
- "Watch someone learn JWT authentication in real time"
- "Debugging score: 2.1 → 4.6 in 8 sessions"
- "The moment it clicks"

**Content ideas:**
- Anonymized teaching transcripts that show the "aha moment"
- Score progression charts (before/after)
- Student testimonials (once we have them)
- "Most common mistake in Module 3" — educational content

## Platform-Specific Notes

### LinkedIn
- Primary platform. All pillars apply.
- Longer text posts (900-1500 chars)
- Videos: 30-60 seconds, terminal aesthetic
- Target: engineering leaders, college admins, hiring managers, students
- Apply 360Brew rules (zero hashtags, save-worthy, natural keywords)

### X (Twitter)
- Shorter, punchier
- Thread format for architecture deep-dives
- GIF/short video clips from Remotion
- Target: developer community, indie hackers, AI builders
- More casual tone, more memes allowed

## Video Content Types (Remotion)

### Type A: Terminal Boot (15s)
Black screen → typing prompt → boot sequence → "Teaching started" → upsk.to

### Type B: Teaching Replay (30s)
Terminal showing AI/student conversation → dimension scores appear → "Evidence, not grades" → upsk.to

### Type C: Feature Demo (45s)
Problem stated → AI teaches adaptively → score generated with evidence → "One prompt. Six skills." → upsk.to

### Type D: Comparison (30s)
Split screen: Traditional bootcamp (slow, generic) vs upsk.to (adaptive, evidence) → "The bootcamp that adapts to you"
